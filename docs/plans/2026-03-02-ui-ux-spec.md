# Cahier des Charges UI/UX — Chrono-Macron

**Date** : 2026-03-02
**Statut** : Validé

---

## 1. Vision Artistique : Le Monde 3D

L'esthétique choisie est le **"Clay Render"** (rendu argile / géométrique doux), fortement inspirée des travaux de **Bruno Simon**. L'objectif est de rendre la donnée économique et politique palpable, non-anxiogène et neutre.

- **Rendu Physique & Doux** : Formes géométriques simples (low-poly) avec bords légèrement arrondis (bevel) et textures mates rappelant la pâte à modeler ou le plastique doux.
- **Éclairage Atmosphérique** : La lumière dicte l'émotion de la période.
  - 2017 (Optimisme) : Lumière chaude, ombres douces, couleurs pastel (vert, jaune doux).
  - 2020 (Crise COVID) : Brume, éclairage froid, teintes gris-bleu.
- **Lisibilité avant tout** : Esthétique épurée pour que les panneaux de données (UI) ressortent sans surcharger l'œil.

---

## 2. Structure du Niveau : Le Chemin & Les Portes

La chronologie n'est **pas un monde ouvert**, mais une **expérience narrative guidée**.

### A. Le Chemin Principal (La Macro Timeline)

- **Composant** : `Timeline.tsx`
- **Design** : La caméra avance sur un chemin géométrique texturé (façon pâte à modeler) composé d'une grille de **4 blocs de large**. Structure rassurante.
- **Environnement** : Sur les bas-côtés, le décor évolue procéduralement (arbres lisses en période faste, blocs fracturés lors des crises).

### B. Les Portes (Zoom Événementiel)

- **Composant** : `EventRoom.tsx`
- **Concept** : Les événements majeurs sont des **lieux physiques** cliquables, thématisés, placés directement sur le chemin.
  - **Gilets Jaunes** : Péage stylisé avec ronds-points miniatures
  - **COVID** : Structure hospitalière entourée de formes virales flottantes
  - **Retraites** : (à définir)
- **Interaction** : Franchir la porte isole l'utilisateur dans une "Data-Room" dédiée.

---

## 3. Mécaniques d'Exploration (UX)

L'utilisateur dicte le rythme de l'histoire, la donnée suit.

- **Scroll / Drag (Sur le chemin)** : Navigation temporelle via molette ou curseur vertical (façon 100,000 Stars). Mouvement de caméra fluide ancré sur un rail.
- **Clic (Sur une Porte)** : Transition caméra qui "rentre" dans l'événement. Scroll temporairement bloqué.
- **Joystick Tactile / Clavier (Dans une Room)** : Contrôleur virtuel (mobile) ou clavier pour explorer les graphiques 3D.

---

## 4. Interface Utilisateur (UI Overlay)

L'interface web classique (HTML/CSS) se superpose au canevas 3D via Tailwind CSS et `<Html>` de Drei.

### A. Design System

| Élément | Choix |
|---------|-------|
| **Typo Titres** | Outfit (géométrique, moderne, impactant) |
| **Typo Data/Corps** | Inter (hyper lisible, neutre, parfaite pour les chiffres) |
| **Style Principal** | Glassmorphism (verre dépoli) |
| **Fond panneaux** | `rgba(255, 255, 255, 0.65)` + `backdrop-filter: blur(12px)` |

### B. Timeline Latérale

- Barre verticale très fine, située **à droite** de l'écran
- Curseur interactif indiquant l'année actuelle (ex: 2020)
- L'utilisateur peut attraper le curseur pour "voyager" rapidement

### C. Panneaux de Données (Recharts + MCP)

- Graphiques alimentés par data.gouv.fr dans des panneaux **Glassmorphism**
- Encart de texte narratif expliquant les anomalies
- **Sources** (INSEE, DGFIP) systématiquement citées en bas de chaque panneau

## 5. Synthèse Technique UI/3D

Le rendu final : illusion parfaite que l'interface Next.js (DOM) et la scène 3D (WebGL) ne font qu'un.

```
<body>              → Fond gris/bleu neutre
<Canvas> (R3F)      → Timeline, lumières, modèles low-poly    (z-index: 1)
<Html> (Drei)       → Panneaux Recharts Glassmorphism          (z-index: 10)
<UI Overlay>        → Barre latérale navigation, fixe          (z-index: 50)
```
